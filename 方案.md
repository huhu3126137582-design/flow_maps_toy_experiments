# Flow Maps on Checkerboard: Toy Experiments 方案

本文档基于 Sander Dieleman 的文章 *Learning the integral of a diffusion model*：
https://sander.ai/2026/05/06/flow-maps.html

目标不是复现大模型论文，而是在一个二维 checkerboard 数据分布上，用最小代码验证 flow map 的核心思想：普通 diffusion / flow matching 每次预测局部速度，然后用很多小步积分；flow map 直接学习这个积分，可以从任意源时间 `s` 跳到任意目标时间 `t`。

文章采用的时间约定是：

- `t = 0`：数据分布。
- `t = 1`：标准高斯噪声。
- 线性插值噪声日程：

```text
x_t = (1 - t) x_0 + t eps,   eps ~ N(0, I)
```

对一对真实样本 `x_0` 和噪声 `eps`，conditional velocity 是：

```text
u = eps - x_0
```

普通 velocity model 学 `v(x_t, t)`；flow map 学：

```text
F(x_s, s, t) = x_t
```

更推荐在 toy experiment 中使用平均速度参数化：

```text
F(x_s, s, t) = x_s + (t - s) V(x_s, s, t)
```

当 `s = t` 时，`V(x_t, t, t)` 退化成普通 instantaneous velocity `v(x_t, t)`。

## 阶段 0：数据、模型和可视化基线

目的：先把 checkerboard 数据、训练循环、采样图、指标都搭好，后面所有 flow map 方法共用这套基础设施。

### 0.1 Checkerboard 数据分布

选择二维 checkerboard，是因为生成质量好坏非常直观：

- 好样本应该落在有效格子中。
- 坏样本会跑到空白格。
- mode dropping 会表现为某些格子缺失。
- 生成模糊会表现为格子边界被抹平。
- 一步、两步、多步采样质量差距肉眼可见。

推荐使用 **显式格子采样** 版本：先采样合法格子 index，再在格子内部均匀采样。这样八个 mode 完全分离，`valid_cell_rate` 和 `mode_coverage` 定义都很干净：

```python
def sample_checkerboard(n, device):
    # 4x4 网格内取 (i, j) 满足 (i + j) % 2 == 0 的 8 个合法格子
    grid_size = 4
    cells = [(i, j) for i in range(grid_size) for j in range(grid_size)
             if (i + j) % 2 == 0]
    idx = torch.randint(0, len(cells), (n,), device=device)
    cells_t = torch.tensor(cells, device=device, dtype=torch.float32)
    base = cells_t[idx]                       # 格子左下角，(n, 2) in [0, grid_size)
    offset = torch.rand(n, 2, device=device)  # 格内均匀
    x = base + offset                          # (n, 2) in [0, grid_size)
    # 标准化到 [-1, 1]
    x = x / (grid_size / 2.0) - 1.0
    return x
```

合法格子集合（中心点）可以保存下来，用于后续指标计算：

```python
def cell_id(x, grid_size=4):
    # 输入 x 已经在 [-1, 1] 范围
    g = (x + 1.0) * (grid_size / 2.0)         # 回到 [0, grid_size)
    i = torch.floor(g[:, 0]).clamp(0, grid_size - 1).long()
    j = torch.floor(g[:, 1]).clamp(0, grid_size - 1).long()
    return i, j

def is_valid_cell(i, j):
    return (i + j) % 2 == 0
```

### 0.2 模型结构

用一个小 MLP 即可：

```text
input:  [x_s(2), emb(s), emb(t), emb(t-s)]
output: V(x_s, s, t) in R^2
```

建议：

- hidden dim：128 或 256。
- layers：4 到 6 层。
- activation：SiLU。
- time embedding：Fourier features 或 sinusoidal embedding。
- 输出预测平均速度 `V`，再通过 `F = x_s + (t - s) V` 得到目标点。

最小版本也可以直接输入 `[x_s, s, t, t-s]`，但 Fourier time embedding 通常更稳。

### 0.3 统一工具函数

后续所有阶段都复用：

```text
x_t(x0, eps, t) = (1 - t) x0 + t eps
u(x0, eps) = eps - x0
F_theta(xs, s, t) = xs + (t - s) V_theta(xs, s, t)
```

### 0.4 可视化

每隔固定步数保存：

- 真实 checkerboard 样本散点图。
- 一步生成：`F_theta(eps, 1, 0)`。
- 两步生成：`1 -> 0.5 -> 0`。
- 四步生成：`1 -> 0.75 -> 0.5 -> 0.25 -> 0`。
- trajectory 图：从同一批噪声出发画路径。

### 0.5 指标

不要只看 loss。建议至少记录：

- `valid_cell_rate`：生成点落在合法 checkerboard 格子的比例。
- `empty_cell_mass`：落到非法格子的比例。
- `mode_coverage`：有多少合法格子被覆盖。
- `cell_hist_kl`：合法格子的直方图与真实分布的 KL 或 L1 距离。
- `MMD`：真实样本和生成样本的 RBF MMD。
- `sliced_W2`：sliced 2-Wasserstein 距离。2D 上算起来很便宜（随机投影到若干 1D 方向，分别排序后算 L2），对 mode dropping 和模糊都敏感，是和论文对齐时比较通用的指标。
- `composition_error`：衡量 flow map 自一致性。定义：

```text
err(s, m, t) = || F(F(x_s, s, m), m, t) - F(x_s, s, t) ||^2
```

评估时建议同时报告两个版本，分别揭示不同问题：

- 等分版本 `m = (s + t) / 2`：稳定、好复现，适合训练曲线监控。
- 随机版本 `m ~ Uniform(t, s)`：更全面，能区分"只在等分点 consistent"和"全区间 consistent"。

`s, t` 采样建议在 `[0, 1]` 上随机配对（保证 `t < s`），按相同分布平均。

## 阶段 1：普通 Flow Matching / Diffusion Velocity Baseline

目的：先训练一个普通 velocity model，作为 teacher 和 sanity check。没有这个阶段，后面很难判断 flow map 的错是来自训练方法，还是来自代码本身。

### 1.1 训练目标

采样：

```text
x0 ~ checkerboard
eps ~ N(0, I)
t ~ Uniform(0, 1)
x_t = (1 - t) x0 + t eps
u = eps - x0
```

训练：

```text
L_FM = E || v_phi(x_t, t) - u ||^2
```

这里 `v_phi` 只输入一个时间 `t`，不是 flow map。

### 1.2 采样

从 `x_1 = eps` 出发，解 ODE：

```text
dx/dt = v_phi(x_t, t)
```

因为采样方向是从 `t=1` 到 `t=0`，Euler 更新可以写成：

```text
x_{t-dt} = x_t - dt * v_phi(x_t, t)
```

测试步数：

- 100 step：近似 teacher quality。
- 20 step：实用 baseline。
- 4 step：少步 baseline。
- 1 step：通常会糊，是 flow map 要超越的对象。

### 1.3 预期现象

100 step 应该能生成清楚的 checkerboard；1 step 多半会有明显空白格污染和边界模糊。这个差距正是 flow map 的实验动机。

## 阶段 2：Teacher-Integrated Flow Map Distillation

目的：用阶段 1 的 velocity model 当 teacher，通过高步数积分得到 `F_teacher(x_s, s, t)`，然后监督训练 flow map。这是最容易成功、最适合第一版 toy experiment 的方法。

### 2.1 构造 teacher target

采样：

```text
x0 ~ checkerboard
eps ~ N(0, I)
s ~ Uniform(0, 1)
t ~ Uniform(0, s)    # 只训练从噪声侧到数据侧
x_s = (1 - s) x0 + s eps
```

用 teacher velocity `v_phi` 从 `s` 积分到 `t`：

```text
x_teacher = ODESolve(v_phi, x_s, s -> t)
```

然后训练 flow map：

```text
L_distill_F = || F_theta(x_s, s, t) - stopgrad(x_teacher) ||^2
```

如果使用平均速度参数化：

```text
V_target = (x_teacher - x_s) / (t - s)
L_distill_V = || V_theta(x_s, s, t) - stopgrad(V_target) ||^2
```

### 2.2 推荐实现

先用 Euler teacher，步数取 50 或 100。checkerboard 是 2D，算力负担很小。

训练时混合三类 interval：

- 短 interval：`|s - t| <= 0.1`，帮助模型学局部。
- 中 interval：`|s - t| <= 0.5`。
- 长 interval：`t = 0` 或接近 0，用于一步生成。

建议 curriculum：

```text
前 20%：只训练短 interval
中间 40%：训练短 + 中 interval
最后 40%：加入长 interval 和 t=0
```

### 2.3 对照实验

比较以下采样方式：

- teacher ODE 100 step。
- teacher ODE 20 step。
- teacher ODE 4 step。
- flow map 1 step：`F_theta(eps, 1, 0)`。
- flow map 2 step：`1 -> 0.5 -> 0`。
- flow map 4 step。

### 2.4 预期现象

如果训练成功，flow map 1 step 应该明显优于普通 velocity model 的 1 step；2 step 或 4 step 通常接近 teacher 多步采样。checkerboard 的合法格子会更清楚，空白格污染更少。

## 阶段 3：Consistency Distillation，只学 `t = 0`

目的：实现 consistency model 风格的 partial flow map。它不是完整 `F(x_s, s, t)`，而是只学从任意 `s` 到数据端 `0`：

```text
C_theta(x_s, s) = F_theta(x_s, s, 0)
```

这是最简单的"一步生成"专用方法。

> **它和阶段 2 的关系**：CD 本质上是 flow map 在 `t = 0` 切片上的特例（只学 `F(·, s, 0)`），不是阶段 2 的平行替代。阶段 2 能从任意 `s` 跳到任意 `t`，CD 只能跳到 `t = 0`。把它放在阶段 2 之后，是因为它实现更轻、单步质量上限可能更高，但失去了 compositionality。

### 3.1 训练目标

用 teacher velocity 从 `s` 往数据方向走一小步：

```text
x_{s-delta} = x_s - delta * v_phi(x_s, s)
```

要求从 `x_s` 直接预测终点，和从 `x_{s-delta}` 预测终点一致：

```text
L_CD = || C_theta(x_s, s) - stopgrad(C_ema(x_{s-delta}, s-delta)) ||^2
```

其中 `C_ema` 是 EMA teacher，或者先用同一个网络 stop-gradient 简化。

### 3.2 边界条件

必须加上靠近数据端的约束：

```text
C_theta(x_0, 0) = x_0
```

实践中可以 oversample 很小的 `s`，或者直接加：

```text
L_boundary = || C_theta(x0, 0) - x0 ||^2
```

### 3.3 优点和局限

优点：

- 实现简单。
- 很适合一步生成。
- 可视化效果容易出。

局限：

- 不是完整 flow map，不能自然地从 `s` 跳到任意 `t`。
- 多步采样时通常需要重新加噪或设计额外过程。
- 对 checkerboard 来说，它能回答“一步好不好”，但不能完整测试 compositionality。

## 阶段 4：Shortcut / Compositional Flow Map

目的：不用显式求导，不碰 JVP，用 compositionality 训练完整 flow map。这是文章里三类 consistency 中最适合 toy code 的路线之一。

核心一致性：

```text
F(F(x_s, s, m), m, t) = F(x_s, s, t)
```

用平均速度写成两个短区间组成一个长区间，训练会更稳定。

### 4.1 Equal-Split Shortcut Loss

采样 `s > m > t`，先从简单的等长切分开始：

```text
m = (s + t) / 2
```

两步 target：

```text
x_m_hat = F_theta(x_s, s, m)
x_t_two = F_theta(x_m_hat, m, t)
```

一大步 prediction：

```text
x_t_one = F_theta(x_s, s, t)
```

训练：

```text
L_shortcut = || x_t_one - stopgrad(x_t_two) ||^2
```

平均速度版本：

```text
V_long = V_theta(x_s, s, t)
V_1 = V_theta(x_s, s, m)
x_m_hat = x_s + (m - s) V_1
V_2 = V_theta(x_m_hat, m, t)
V_target = ((m - s) V_1 + (t - m) V_2) / (t - s)
L_shortcut_V = || V_long - stopgrad(V_target) ||^2
```

### 4.2 Grounding Loss

只靠 self-consistency 会有退化解，所以要让 `s = t` 的地方学普通 velocity：

```text
L_ground = || V_theta(x_r, r, r) - (eps - x0) ||^2
```

总损失：

```text
L = L_ground + lambda * L_shortcut_V
```

### 4.3 训练策略

建议：

- 大量 oversample `s = t`，保证模型先像一个正常 velocity model。
- 先训练 `L_ground` 若干步，相当于 warmup。
- 然后加入小 interval 的 shortcut。
- 最后逐渐增加 `|s - t|` 到 1。

curriculum 示例：

```text
0 - 10k steps:   only L_ground
10k - 30k:       max |s-t| = 0.125
30k - 60k:       max |s-t| = 0.25
60k - 100k:      max |s-t| = 0.5
100k+:           max |s-t| = 1.0
```

### 4.4 预期现象

这个阶段可以重点看：

- `composition_error` 是否下降。
- 1 step 是否逐步变清楚。
- 2 step / 4 step 是否比 1 step 更稳。
- 长 interval 训练过早时是否出现塌缩。

如果出现所有点都往中心塌，通常是 grounding 不够强、EMA target 不稳，或者长 interval 上得太早。

## 阶段 5：MeanFlow From Scratch

目的：复现文章中最重要的 from-scratch 思路之一：不需要预训练 teacher，用 marginal-from-conditional learning 直接训练平均速度 flow map。

MeanFlow 基于 Eulerian consistency。平均速度形式为：

```text
V(x_s, s, t)
= v(x_s, s)
  + (t - s) [ dV/ds + grad_x V · v(x_s, s) ]
```

从 scratch 时，把 **所有出现的 marginal velocity `v(x_s, s)` 都替换成 conditional velocity** `u = eps - x0`，包括 JVP 里的 tangent 方向：

```text
u = eps - x0
```

替换后的等式：

```text
V(x_s, s, t)
≈ u
  + (t - s) [ dV/ds + grad_x V · u ]
```

注意这里第二项中 `grad_x V` 点乘的也是 `u`，不是别的——这正是 marginal-from-conditional 的关键：JVP 的 tangent 必须沿 `u` 方向，否则恒等式不再成立。

### 5.1 MeanFlow Loss

采样：

```text
x0 ~ checkerboard
eps ~ N(0, I)
s ~ Uniform(0, 1)
t ~ Uniform(0, s)
x_s = (1 - s) x0 + s eps
u = eps - x0
```

计算 JVP（tangent 向量 `(u, 1, 0)`，即对 `x_s` 沿 `u` 方向、对 `s` 单位、对 `t` 零）：

```text
jvp = d/dalpha V(x_s + alpha * u, s + alpha, t) | alpha=0
    = grad_x V · u + dV/ds
```

target：

```text
target = u + (t - s) * jvp
```

loss：

```text
L_MF = || V_theta(x_s, s, t) - stopgrad(target) ||^2
```

### 5.2 PyTorch 实现路线

优先使用：

```python
torch.func.jvp
```

输入 tangent：

```text
for x_s: u
for s:   1
for t:   0
```

如果先想写最小版本，可以用 finite difference：

```text
jvp ≈ [V(x_s + h u, s + h, t) - V(x_s, s, t)] / h
```

建议 `h = 1e-3` 到 `1e-2`，不要太小，否则浮点误差明显。

### 5.3 稳定性技巧

MeanFlow 对时间采样敏感。建议：

- oversample `s = t`，例如 25% batch 用 `t = s`。
- 对 `|t - s|` 做 curriculum。
- 当 `|t - s|` 太小时，loss 权重不要爆炸。
- 使用 EMA 参数做采样评估。
- 可以用 Huber / pseudo-Huber 降低 outlier 影响。

### 5.4 预期现象

如果成功，它不需要阶段 1 的 teacher，也可以学出一步或少步 checkerboard。和阶段 2 的 teacher distillation 相比，训练更敏感，但研究价值更高。

## 阶段 6：Improved MeanFlow (iMF)

目的：实现文章中更优雅、更低方差的 MeanFlow 变体。它把 flow map 参数化成一个普通 flow matching predictor 来训练。

iMF 定义（**注意**：第二项里 `grad_x V` 点乘的不是 `V(x_s, s, s)`，而是 conditional velocity `u = eps - x0`，和 MeanFlow 一致）：

```text
W(x_s, s, t)
= V(x_s, s, t)
  - (t - s) * stopgrad[ dV/ds + grad_x V · u ]
```

实现上，把 stopgrad 内部那一项直接写成 tangent `(u, 1, 0)` 的 JVP：

```text
jvp = jvp_of_V_at(x_s, s, t)  with tangent (u, 1, 0)
W   = V_theta(x_s, s, t) - (t - s) * stopgrad(jvp)
```

然后用普通 flow matching loss：

```text
L_iMF = || W(x_s, s, t) - (eps - x0) ||^2
```

### 6.1 和 MeanFlow 的区别

MeanFlow 是让 `V` 拟合一个 stop-gradient target：

```text
V ≈ u + correction
```

iMF 是把 `V` 和 correction 组合成 `W`，让 `W` 拟合 conditional velocity：

```text
W ≈ u
```

文章中强调 iMF 往往方差更低，形式也更像普通 diffusion / flow matching 训练。

### 6.2 Toy 实验对照

在 checkerboard 上比较：

- MeanFlow。
- improved MeanFlow。
- Shortcut from scratch。
- Teacher distillation flow map。

重点看：

- 训练 loss 是否稳定。
- 一步样本是否清楚。
- mode coverage。
- composition error。
- 对不同随机种子的敏感程度。

## 阶段 7：Lagrangian / Eulerian Distillation 对照

目的：不是为了最先成功，而是为了理解文章中的三种 consistency 视角。这个阶段可以作为加分实验。

### 7.1 Lagrangian Consistency

Lagrangian 看目标时间 `t` 的变化：

```text
dF(x_s, s, t) / dt = v(F(x_s, s, t), t)
```

loss：

```text
L_L = || dF_theta/dt - v_phi(F_theta(x_s, s, t), t) ||^2
```

其中 `v_phi` 是阶段 1 的 teacher velocity。

### 7.2 Eulerian Consistency

Eulerian 看源时间 `s` 的变化：

```text
dF/ds + grad_x F · v_phi(x_s, s) = 0
```

loss：

```text
L_E = || dF_theta/ds + grad_x F_theta · v_phi(x_s, s) ||^2
```

### 7.3 From-scratch 变体（LSD / ESD）

上面 7.1 / 7.2 都是带 teacher `v_phi` 的 distillation 版本（LMD / EMD）。文章里还有 self-distillation 版本，把 `v_phi` 替换成自身的 `V_theta(x, s, s)` 或 conditional `u`，无需 teacher：

**LSD（Lagrangian Self-Distillation）**：

```text
L_LSD = || dF_theta/dt - V_theta(F_theta(x_s, s, t), t, t) ||^2
```

或更稳的 conditional 版本（要求 t 接近 s 时退化为 conditional flow）：

```text
L_LSD_cond = || dF_theta/dt|_{t=s} - u ||^2  + 一致性正则
```

**ESD（Eulerian Self-Distillation）**：

```text
L_ESD = || dF/ds + grad_x F · V_theta(x_s, s, s) ||^2
```

实现细节比 LMD / EMD 难一档：需要 stop-gradient 阻断高阶导数、EMA target 网络稳住自参考。

### 7.4 实验价值

这个阶段可以回答：

- JVP 版本和 finite difference 版本哪个更稳。
- stop-gradient 放在哪里更重要。
- Lagrangian 和 Eulerian 在 checkerboard 这种多模态分布上谁更容易训练。
- 是否需要 teacher：LMD/EMD vs LSD/ESD 在收敛速度、最终质量上的差距。

建议在已经完成阶段 2 或阶段 4 后再做。

## 阶段 8：Flow Map Look-Ahead Steering

目的：验证文章里提到的 flow map 额外用途：它不仅能加速采样，还能提供一个 clean、可微的 look-ahead，用于 reward steering。

### 8.1 定义 toy reward

在 checkerboard 上定义一个偏好，例如：

```text
R(x0) = indicator / smooth score for top-right valid cells
```

或者：

```text
R(x0) = - distance_to_target_cell_center
```

### 8.2 采样时 steering

在中间噪声状态 `x_s`，用 flow map 预测终点：

```text
x0_hat = F_theta(x_s, s, 0)
reward = R(x0_hat)
```

对 `x_s` 求梯度：

```text
g = grad_{x_s} reward
x_s <- x_s + eta * g
```

然后继续用 flow map 或 velocity model 采样。

### 8.3 对照

比较：

- 无 steering。
- 只用普通 denoiser 的 one-step clean estimate 做 steering。
- 用 flow map look-ahead 做 steering。
- 直接大量采样后 rejection / filtering。

预期：flow map look-ahead 的 reward 梯度更干净，生成点更容易集中到目标格子，同时不至于完全跑出 checkerboard 分布。

## 推荐执行顺序

如果目标是最快得到有说服力的 toy results：

```text
阶段 0：checkerboard + 可视化 + 指标
阶段 1：普通 flow matching baseline
阶段 2：teacher-integrated flow map distillation
阶段 4：shortcut / compositional flow map
阶段 5：MeanFlow
阶段 6：iMF
阶段 8：look-ahead steering
```

阶段 3 很适合快速做一步生成，但不是完整 flow map。阶段 7 更适合理解理论和做 ablation。

## 实现注意事项

1. 时间方向要始终统一：本文档使用 `t=0` 数据、`t=1` 噪声，采样方向是 `1 -> 0`。
2. `F = x_s + (t - s) V` 中，采样时 `t - s` 是负数，这是正常的。
3. checkerboard 的尺度要和高斯噪声尺度匹配。建议把数据标准化到大约 `[-1, 1]`。
4. flow map 的长 interval 比短 interval 难很多，不要一开始就强迫模型学 `1 -> 0`。
5. `s = t` grounding 很关键，否则 self-consistency 容易出现退化解。
6. MeanFlow / iMF 中 JVP 的 stop-gradient 细节会显著影响训练稳定性。
7. 评估时不要只看 one-step；同时看 1、2、4、8 step，因为 flow map 近似误差可以通过多步缓解。

---

## 一句话总结

最稳路线是：先用普通 flow matching 在 checkerboard 上训练一个多步 teacher，再把它 distill 成 `V_theta(x_s, s, t)` flow map，验证一步和少步生成；随后用 shortcut、MeanFlow、iMF 做 from-scratch 对照，最后用 flow map 的 clean look-ahead 做 reward steering。checkerboard 会非常清楚地暴露每种方法的 mode dropping、空白格污染和边界模糊问题。
