## 0. 全局约定与公共基础设施

### 0.1 记号约定

- 时间方向：`t=0` 是数据，`t=1` 是纯噪声。
- 线性随机插值：
  \[
  x_t=(1-t)x_0+t\varepsilon,\qquad \varepsilon\sim\mathcal N(0,I_2).
  \]
- 条件速度：
  \[
  v_{\mathrm{cond}}=\varepsilon-x_0.
  \]
- 边缘速度：
  \[
  v(x_t,t)=\mathbb E[\varepsilon-x_0\mid x_t].
  \]
- 训练和采样使用 `0 <= t <= s <= 1`，其中 `s` 是起点时间，`t` 是终点时间。
- Flow map 使用平均速度参数化：
  \[
  F_\theta(x,s,t)=x+(t-s)V_\theta(x,s,t).
  \]
- 对角边界为：
  \[
  V_\theta(x,t,t)=v_\theta(x,t).
  \]
- 生成过程为：
  \[
  x_0=F_\theta(\varepsilon,1,0).
  \]

### 0.2 网络架构

- 所有方法使用同一个 `FlowMapNet`。
- 输入为 `concat(x, emb(s), emb(t))`。
- 主干为 6 个隐藏层，每层 512 维，激活函数为 SiLU，输出为 2 维平均速度。
- `s` 和 `t` 分别使用 64 维 Gaussian Fourier features，频率尺度固定为 16，再经过独立线性投影。
- Teacher 训练时始终输入 `s=t`，其输出解释为瞬时速度。
- Teacher 蒸馏阶段使用 teacher 权重初始化 student。
- 所有模型使用 decay 为 `0.999` 的 EMA，评估只使用 EMA 权重。

### 0.3 训练配置

- 每个训练 batch 在线调用 `sample_checkerboard` 生成数据。
- 在线采样器持有持续更新的 `torch.Generator`，不会在每个 batch 重置随机种子。
- 优化器为 Adam。
- 各方法的 batch size、训练步数、学习率和时间采样策略在对应章节中定义。
- 所有方向导数使用 `torch.func.jvp`。
- 涉及 JVP 的训练目标按各方法定义保留或停止对应分支的梯度。
- SG-LMD 主训练运行到 70k；随后从 60k checkpoint 确定性重放到
  75k，并在 65k–75k 范围内每 1k 比较一次。batch size 为 32,768，
  学习率按 90k 日程从 `3e-4` 余弦衰减到 `3e-5`；SG target
  停止梯度，JVP 分支保留反向传播。

## 阶段零：评估指标体系

Checkerboard 使用 `bounds=(-1,1)` 和 4x4 网格，共有 8 个深色格子。每格面积为 0.25，支撑集总面积为 2，支撑集内真实密度为 0.5。

### 0.1 Checkerboard 专属指标

1. **In-support 率**

   判断生成样本是否位于深色格子：

   ```python
   in_bounds = ((x >= -1) & (x <= 1)).all(dim=-1)
   cell = torch.floor((x + 1) / 0.5).long().clamp(0, 3)
   is_dark = ((cell.sum(dim=-1) % 2) == 0) & in_bounds
   ```

2. **离支撑集距离**

   对支撑集外样本计算其到最近深色格子的欧氏距离，报告均值和 95% 分位数。

3. **模式覆盖**

   统计 8 个深色格子的样本比例，报告占比超过 1% 的格子数，以及格子占比分布相对均匀分布的 KL 散度。

4. **格内均匀性**

   将每个深色格子划分为 8x8 子网格，计算生成样本子网格分布相对均匀分布的 Jensen-Shannon 散度。

### 0.2 通用分布指标

5. **二维直方图距离**

   在 `[-1.2,1.2]^2` 上使用 96x96 网格，比较生成直方图与解析真实分布，报告 Total Variation 和 Jensen-Shannon 散度。区间外样本作为独立 overflow bin 计入。

6. **Sliced Wasserstein-2**

   使用固定的 200k 真实参考样本和 512 个固定随机投影方向计算 SW2。

7. **MMD**

   使用 RBF 核，带宽固定为 `{0.05, 0.1, 0.2, 0.5}`，报告四个带宽结果的平均值。

### 0.3 效率与诊断

8. **NFE 评估**

   Teacher checkpoint 选择固定使用 Euler NFE=128；LMD checkpoint
   选择评估 NFE 为 `{1,2,4}`，最终可视化额外展示 NFE=8。
   Teacher 每个 Euler 步计 1 NFE；flow map 每个等距时间分段调用一次
   Student 并计 1 NFE。

9. **统一可视化**

   - 真实分布与 50k 生成样本的并排散点图；
   - 真实分布与生成分布的 96×96 二维直方图；
   - 所有直方图面板共享线性颜色范围。

10. **一致性诊断**

    组合误差：
    \[
    \left\|F_\theta(F_\theta(x_1,1,0.5),0.5,0)
    -F_\theta(x_1,1,0)\right\|_2.
    \]

    边界误差：
    \[
    \left\|V_\theta(x_t,t,t)-v_{\mathrm{teacher}}(x_t,t)\right\|_2.
    \]

### 0.4 评估协议

- 每种方法使用 1 个固定训练种子，默认 `seed=0`。
- 每种方法使用同一组 50k 固定评估噪声。
- 最终指标报告单次训练结果，不计算跨种子的均值和标准差。
- Teacher 每 15k steps 使用固定 10k 噪声评估并保存 checkpoint。
- LMD 主训练每 5k steps 使用固定 10k 噪声评估，常规 checkpoint
  间隔为 10k；细搜重放阶段 checkpoint 间隔为 1k。
- Teacher 最终导出 120k EMA；LMD 最终直接采用 72k checkpoint 中的 EMA。
- 指标模块先验证真实样本对真实样本、Gaussian 噪声对真实样本两组基线。

## 阶段一：Teacher - Flow Matching

### 目标

训练边缘速度场 \(v_\phi(x,t)\)，作为 SG-LMD 的冻结 Teacher 和
Euler ODE 采样基线。最终采用
`outputs/teacher/teacher.pt` 中的 120k EMA 权重。

### 训练目标

\[
\mathcal L_{\mathrm{FM}}
=
\mathbb E_{x_0,\varepsilon,t}
\left[
\left\|
V_\phi(x_t,t,t)-(\varepsilon-x_0)
\right\|_2^2
\right],
\qquad t\sim\mathcal U(0,1).
\]

Teacher 使用公共 `FlowMapNet`，训练时固定 `s=t`。

### 实际训练配置

- 实际训练到 120k steps，固定 `seed=0`。
- batch size 为 65,536。
- 优化器为 Adam，初始学习率为 `1e-3`。
- 学习率按 120k 步余弦衰减到 `1e-4`。
- 全局梯度范数裁剪阈值为 10。
- EMA decay 为 `0.999`，每个优化器 step 后更新 EMA。
- 训练和损失计算使用 FP32；CUDA 上允许 TF32。
- 每 15k steps 保存训练 checkpoint，并用固定 10k 噪声、
  Euler NFE=128 写入一次轻量评估。
- checkpoint 完整保存模型、优化器、EMA、训练 step、累计耗时和随机状态，
  支持 `--resume auto` 确定性恢复。

### Checkpoint 选择

训练后使用同一批固定 50k 噪声比较 90k、105k 和 120k 的 EMA，
采样器统一使用 Euler NFE=128：

| Step | In-support | Histogram TV | Histogram JSD |
|---:|---:|---:|---:|
| 90k | 0.99766 | 0.10538 | 0.00950 |
| 105k | 0.99806 | 0.10461 | 0.00932 |
| **120k** | **0.99858** | **0.10465** | **0.00919** |

120k 的 in-support 和 JSD 最优，因此不再继续训练，导出其 EMA 为
`teacher.pt`。

## 阶段二：Teacher 蒸馏

Teacher 参数在蒸馏期间冻结。Student 使用 `teacher.pt` 初始化。
Teacher 蒸馏阶段采用 SG-LMD。

### 2.1 Lagrangian Map Distillation

本节实际采用带停止梯度的 Lagrangian Map Distillation（SG-LMD），
仅学习 `0 <= t <= s <= 1` 的噪声到数据方向。Teacher 固定为阶段一
选出的 120k EMA，最终采用
`outputs/lmd/checkpoint_072000.pt` 中的 Student EMA。

#### 训练目标

构造：

\[
x_s=(1-s)x_0+s\varepsilon.
\]

训练损失只使用 SG-LMD：

\[
\mathcal L_{\mathrm{SG-LMD}}
=
\mathbb E
\left[
\left\|
\partial_tF_\theta(x_s,s,t)
-
v_\phi\left(
\operatorname{sg}\left[F_\theta(x_s,s,t)\right],
t
\right)
\right\|_2^2
\right].
\]

其中：

\[
F_\theta(x_s,s,t)
=
x_s+(t-s)V_\theta(x_s,s,t),
\]

\[
\partial_tF_\theta
=
V_\theta+(t-s)\partial_tV_\theta.
\]

对角样本满足：

\[
\partial_tF_\theta(x_t,t,t)=V_\theta(x_t,t,t).
\]

对角样本和非对角样本统一计算
\(\mathcal L_{\mathrm{SG-LMD}}\)，不增加独立对角损失、额外损失权重或稳健损失。
逐样本损失为二维残差平方和，训练损失为 batch 均值。

#### 梯度实现

- Teacher 加载 `outputs/teacher/teacher.pt`，设置为 `eval()` 并冻结全部参数。
- Student 使用 Teacher 权重初始化。
- 使用 `torch.func.jvp` 对终点时间 `t` 求方向导数。
- Teacher 输入使用 `F_theta(...).detach()`，实现
  \(\operatorname{sg}[F_\theta]\)。
- Teacher 前向不建立计算图，Teacher 参数不产生梯度。
- `dt_flow` 保留计算图，梯度通过 JVP 分支更新 Student。

核心计算为：

```python
mapped, dt_flow = torch.func.jvp(
    lambda target_t: student.flow(x_s, s, target_t),
    (t,),
    (torch.ones_like(t),),
)

with torch.no_grad():
    target_velocity = teacher(mapped.detach(), t, t)

loss = (
    (dt_flow - target_velocity)
    .square()
    .sum(dim=-1)
    .mean()
)
```

#### 实际训练配置

- 固定训练种子为 `seed=0`。
- batch size 为 32,768；为了控制 JVP 显存，按 2,048
  的 microbatch 累积梯度。
- 优化器为 Adam。
- 学习率使用 90k 日程，从 `3e-4` 余弦衰减到 `3e-5`。
- 全局梯度范数裁剪阈值为 5。
- EMA decay 为 `0.999`，每个优化器 step 后更新 EMA。
- 训练和损失计算使用 FP32；CUDA 上允许 TF32。
- 输出统一存放在 `outputs/lmd/`。
- 训练期评估、细搜和最终可视化均只使用 Student EMA。

#### 对角比例调度

训练 step 记为 \(k\)，对角样本比例为：

\[
p_{\mathrm{diag}}(k)=
\begin{cases}
0.10, & k<5000,\\
0.10+0.15\dfrac{k-5000}{25000},
& 5000\le k<30000,\\
0.25, & k\ge30000.
\end{cases}
\]

对应配置为：

| 训练区间 | 对角比例 | 非对角比例 |
|---|---:|---:|
| 0–5k | 10% | 90% |
| 5–30k | 从 10% 线性增加到 25% | 从 90% 线性下降到 75% |
| 30k 以后 | 25% | 75% |

每步对角样本数为 `round(batch_size * p_diag)`，其余为非对角样本。
对角样本独立采样：

\[
\tau\sim\mathcal U(0,1),\qquad s=t=\tau.
\]

#### 非对角时间课程

最大时间间隔为：

\[
d_{\max}(k)=
\begin{cases}
0.1, & k<5000,\\
0.1+0.9\dfrac{k-5000}{25000},
& 5000\le k<30000,\\
1.0, & k\ge30000.
\end{cases}
\]

非对角时间对在以下截断三角域内按面积均匀采样：

\[
0<s-t\le d_{\max},\qquad 0\le t<s\le1.
\]

令 \(u,r\sim\mathcal U(0,1)\)，时间间隔 \(g=s-t\) 使用解析逆 CDF
采样：

\[
g
=
1-\sqrt{1-u(2d_{\max}-d_{\max}^2)},
\]

\[
t=r(1-g),\qquad s=t+g.
\]

## 阶段三：自蒸馏

LSD 从随机初始化开始训练，不使用预训练 Teacher。

### 3.1 Lagrangian Self-Distillation

LSD 不将阶段一 Teacher 替换为另一个冻结模型，而是在同一个随机初始化
的 `FlowMapNet` 中联合训练对角瞬时速度场和非对角 Flow Map。对角分支
通过 Flow Matching 提供外部监督，非对角分支将当前瞬时速度场作为
停止梯度的 instantaneous Teacher，自蒸馏成长距离 Flow Map。训练器
不提供 `--teacher` 参数，也不读取阶段一或阶段二的任何 checkpoint。

#### 训练目标

每个逻辑 batch 固定划分为 75% 对角样本和 25% 非对角样本。对角分支
独立采样：

\[
\tau\sim\mathcal U(0,1),\qquad
x_\tau=(1-\tau)x_0+\tau\varepsilon,
\]

并使用标准 Flow Matching 损失：

\[
\mathcal L_{\mathrm{FM}}
=
\mathbb E
\left[
\left\|
V_\theta(x_\tau,\tau,\tau)-(\varepsilon-x_0)
\right\|_2^2
\right].
\]

非对角分支在完整三角域
\(0\le t<s\le1\) 上按面积均匀采样。令
\(u_1,u_2\sim\mathcal U(0,1)\)，使用：

\[
s=\max(u_1,u_2),\qquad t=\min(u_1,u_2),
\]

\[
x_s=(1-s)x_0+s\varepsilon,
\qquad
F_\theta(x_s,s,t)
=x_s+(t-s)V_\theta(x_s,s,t).
\]

LSD 损失为：

\[
\mathcal L_{\mathrm{LSD}}
=
\mathbb E
\left[
\left\|
\partial_tF_\theta(x_s,s,t)
-
\operatorname{sg}
\left[
V_\theta\left(
\operatorname{sg}[F_\theta(x_s,s,t)],t,t
\right)
\right]
\right\|_2^2
\right].
\]

总损失按整个逻辑 batch 的逐样本损失取均值：

\[
\mathcal L
=
0.75\,\mathcal L_{\mathrm{FM}}
+
0.25\,\mathcal L_{\mathrm{LSD}}.
\]

逐样本损失均为二维残差平方和。不使用两个 branch mean 直接相加的
equal-branch 版本，不增加 Huber loss、额外损失系数或 learned
loss-weight head。

#### 梯度实现

对角分支直接调用 Student 计算瞬时速度，不经过 JVP。非对角分支使用
`torch.func.jvp` 对终点时间 \(t\) 求方向导数：

```python
mapped, dt_flow = torch.func.jvp(
    lambda target_t: student.flow(x_s, s, target_t),
    (t,),
    (torch.ones_like(t),),
)

with torch.no_grad():
    target_velocity = student(mapped.detach(), t, t)

off_loss = (
    (dt_flow - target_velocity)
    .square()
    .sum(dim=-1)
)
```

梯度规则固定如下：

- `dt_flow` 保留计算图，梯度通过 JVP 分支更新 Student。
- `mapped.detach()` 阻断 transported point 到 target 的梯度路径。
- 对角 Teacher velocity 在 `torch.no_grad()` 中使用当前 Student 参数
  计算，不使用单独 Teacher 模型。
- instantaneous Teacher 仅作为停止梯度 target；当前优化 step 更新前后
  不复制模型参数。
- EMA 只用于评估、checkpoint 选择和最终分布生成，不参与训练 target。

#### 实际训练配置

- 固定训练种子为 `seed=0`。
- 逻辑 batch size 为 100,000，其中固定包含 75,000 个对角样本和
  25,000 个非对角样本。
- 物理 `microbatch-size=2048`，通过梯度累积完成一个逻辑 batch。
- 分块反向时，每个 microbatch 使用其逐样本损失之和除以 100,000，
  从而与完整 batch 的损失和梯度严格等价。
- 使用 Adam，初始学习率为 `1e-3`。
- 前 35k steps 学习率保持 `1e-3`，之后使用平方根衰减：

  \[
  \operatorname{lr}(k)
  =
  \frac{10^{-3}}
  {\sqrt{\max(k/35000,1)}}.
  \]

- 全局梯度范数裁剪阈值为 10。
- EMA decay 为 `0.999`，每个优化器 step 后更新 EMA。
- 使用 FP32，CUDA 上允许 TF32。
- 不进行对角 warmup、最大 gap 课程或 Flow Matching 预训练；从第一个
  step 起使用固定 75%/25% 划分和完整三角域。
- 正式候选范围截止到 130k。原始运行继续到 135,817，并在尝试第
  135,818 步时因梯度范数非有限而停止；该不稳定尾部不进入选择。
- 输出统一存放在 `outputs/lsd/`。

对应 CLI 默认值为：

```text
--output-dir outputs/lsd
--steps 130000
--batch-size 100000
--microbatch-size 2048
--diag-fraction 0.75
--learning-rate 1e-3
--lr-decay-steps 35000
--grad-clip 10
--ema-decay 0.999
--seed 0
--resume auto
```

#### Pilot、训练监控与恢复

1. 先运行 2k-step pilot，再从 pilot checkpoint 继续正式训练。实际
   `pilot_summary.json` 的 finite、FM/LSD 下降、裁剪率和显存稳定检查
   均通过。
2. pilot 和主训练使用相同的 100k 逻辑 batch 与 2048 microbatch。
3. 每 100 steps 写入总损失、FM 损失、LSD 损失、学习率、梯度范数、
   裁剪状态、吞吐量、累计耗时和 CUDA 显存峰值。
4. 每 1k steps 按 \(s-t\) 分桶记录 LSD 逐样本平方损失的 mean、
   median、P95 和 P99，并记录 `dt_flow`、instantaneous target 和
   Student 平均速度范数。
5. gap bucket 沿用 SG-LMD：

   ```text
   (0, 0.05], (0.05, 0.1], (0.1, 0.2],
   (0.2, 0.4], (0.4, 0.6], (0.6, 0.8], (0.8, 1.0]
   ```

6. 每 5k steps 使用固定 10k 噪声监控 EMA 的 NFE=1/2/4，并记录
   composition error 与独立诊断 batch 上的 LSD residual。该在线评估
   不负责最终 checkpoint 选择。
7. 每 10k steps 保存完整 checkpoint；100k 后每 5k steps 额外保存
   checkpoint，直到 130k。
8. checkpoint 保存 Student、Adam、EMA、训练 step、累计耗时、训练
   配置、监控状态和 Python/NumPy/PyTorch/CUDA/自定义 Generator
   随机状态，支持 CPU/CUDA 确定性恢复。
9. `--resume auto` 或指定 checkpoint 恢复后，下一批数据、对角和
    非对角时间、loss 以及优化器更新均可复现。

#### Checkpoint 选择

原始运行曾按 NFE=1 记录在线 best，但最终选择不使用该结果；现行训练
代码也不再维护这条分支，而采用独立的四-NFE 综合筛选：

1. 对 `outputs/lsd/` 中 step 不超过 130k 的全部 checkpoint 使用固定
   10k 噪声，评估 EMA 的 NFE=1/2/4/8。
2. 每个 NFE 分别计算 SW2、histogram JSD 和 in-support rate，共
   12 个 criterion。每项指标在候选间转换为 \([0,1]\) 归一化名次；
   SW2/JSD 越小越好，in-support 越大越好。12 个名次等权平均，
   composite score 越小越好。
3. 粗筛综合前 6 名使用固定 50k 噪声和 200k 真实参考样本复评，最优
   区域落在 100k 附近。
4. 从 `checkpoint_090000.pt` 确定性重放到 105k，每 1k 保存一次。
   重放得到的 100k Student、EMA、Adam 和随机状态与原始 100k
   checkpoint 逐位一致。
5. 对 95k–105k 共 11 个 checkpoint 使用 50k 样本精排。单种子前四名
   为 101k、99k、98k、100k。
6. 对前四名使用种子 20260614、20260615、20260616，各自生成 50k

#### 测试与验收

- 验证 \(F_\theta(x,t,t)=x\)。
- 验证
  \(\partial_tF_\theta(x,t,t)=V_\theta(x,t,t)\)。
- 使用解析函数验证 JVP 数值和 JVP 反向梯度。
- 验证对角分支严格等价于线性插值 Flow Matching。
- 验证 `mapped` target 路径和 instantaneous Teacher target 均无梯度。
- 验证梯度只通过 `dt_flow` 分支更新非对角 Student。
- 验证 100k batch 精确包含 75,000 个对角样本和 25,000 个非对角样本。
- 验证总损失等于所有 100k 样本损失之和除以 100,000。
- 验证完整 batch 与不同 microbatch 分块产生数值一致的损失和梯度。
- 验证三角采样始终满足 \(0\le t<s\le1\)，并用经验矩验证其为完整
  三角域面积均匀分布。
- 验证训练 target 使用当前 Student 而不是 EMA 参数。
- 验证 checkpoint 恢复后下一批数据、时间、loss 和优化器更新可复现。
- 验证 90k 到 105k 的确定性重放中，100k 的 Student、EMA、Adam 和
  随机状态与原始 checkpoint 逐位一致。
- 验证四 NFE 综合排名对 SW2/JSD 使用升序、对 in-support 使用降序，
  并对 12 个 criterion 等权平均。

- 最终报告 99k checkpoint 在三个固定种子上的 NFE=1/2/4/8

## 实现约束

- `common.py` 只放置跨方法共享的随机种子、设备、EMA、学习率和 checkpoint 工具。
- `torch.func.jvp` 的方向导数实现使用解析线性函数编写单元测试。
- Teacher Euler 采样和 flow map 分段采样都严格按网络调用次数统计 NFE。
- 所有方法固定使用相同网络宽度和数据采样器；同类方法使用相同 batch size 和训练步数。
